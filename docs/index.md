# Выбор и обоснование критериев для сравнения асинхронных веб-фреймворков в экосистеме Python

> test autodeploy

## Актуальность темы НИР

Актуальность данной работы обусловлена наличием в экосистеме Python большого количества разнообразных асинхронных веб-фреймворков и растущим спросом на высокопроизводительные веб-приложения.

## Цель НИР

Выбрать и обосновать критерии для сравнения асинхронных веб-фреймворков в экосистеме Python.

## Объект и предмет исследования

**Объект исследования** – подходы к сравнительному анализу серверных веб-фреймворков.

**Предмет исследования** – критерии, инструменты и сценарии объективного сравнения серверных веб-фреймворков.

## Задачи исследования

1. Проанализировать методологии проведения сравнительного анализа серверных веб-фреймворков.
2. Составить обзор подходов к проведению сравнительного анализа серверных веб-фреймворков.
3. Сформировать и обосновать критерии для сравнительного анализа асинхронных веб-фреймворков в экосистеме Python.

## Результаты исследования

1. Для полноценного сравнительного анализа серверных веб-фреймворков необходимо их оценивать по набору характеристик: производительность, популярность, качество, сложность кода и другим.
2. Асинхронные веб-фреймворки в экосистеме Python имеют собственные архитектурные различия, а также могут использовать различные реализации внутренних и внешних компонентов, что влияет на производительность.
3. Существует большое количество инструментов для замера количественных параметров, применимых к асинхронным веб-фреймворкам в экосистеме Python.
4. Apache Jmeter и Radon являются достаточно сбалансированными решениями для оценки замеряемых ими количественных параметров, хорошо зарекомендовавшими себя в исследовательской среде, что делает их подходящими для использования в исследовательском эксперименте.
